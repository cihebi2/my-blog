计算蛋白质设计(Computational Protein Design, CPD)作为一个旨在创造具有特定结构和功能的新型蛋白质的领域，正在经历一场深刻的变革。本报告全面调研了2021年以来发表的广泛认可且非商业化的蛋白质设计工具及其应用。

## I. 现代计算蛋白质设计导论

### A. 蛋白质工程的演进格局

计算蛋白质设计经历了显著的演变。历史上，蛋白质工程主要依赖于对现有蛋白质进行改造。然而，近年来，该领域正经历一场范式转变，逐渐走向**"从头设计"(de novo design)**——即从基本原理出发，创造自然界中不存在的、具有复杂结构和功能的蛋白质。

人工智能(AI)，特别是深度学习(DL)和生成模型，在推动CPD革命中扮演了核心角色。这些方法通过利用海量的蛋白质序列和结构数据库，实现了前所未有的设计能力。AI与生物学的融合并非简单的技术叠加，而是代表了蛋白质设计方法的根本性变革。

例如：
- **RFdiffusion** 等工具的设计灵感来源于AI图像生成器
- **ProteinMPNN** 运用了稳健的深度学习方法进行序列设计
- 成功的AI/ML架构（如图神经网络、Transformer、扩散模型）正被迅速应用于蛋白质特异性任务

更值得注意的是，蛋白质科学领域已经从主要关注"根据序列预测结构"发展到能够"生成具有期望特性的新序列和结构"。这一进步是合乎逻辑的发展：从理解现有蛋白质（预测）到创造新蛋白质（设计）。

### B. 开源工具的重要性

开源工具在推动蛋白质设计领域发展中扮演着至关重要的角色：

- **技术普及化**：使先进的蛋白质设计能力不再局限于少数专业实验室
- **社区驱动**：加速了知识共享和技术迭代
- **学术自由**：为科研人员提供了不受商业限制的探索平台

### C. 本报告的范围与目标

本报告旨在全面调研2021年以后发表的、获得广泛认可且非商业化的蛋白质设计工具，以及蛋白质设计流程中不可或缺的辅助工具。报告将详细阐述这些工具的应用场景和使用方法，为相关领域的研究人员提供一份实用的参考指南。

## II. 广泛认可的蛋白质设计工具（2021年后，非商业化）

| 工具名称 | 主要功能 | 核心创新/原理 | 主要应用领域 | 可用性 | 主要出版物 |
|---------|---------|--------------|-------------|--------|-----------|
| **ProteinMPNN** | 序列设计 | 图神经网络(GNN) | 从头设计、蛋白质重设计、序列库生成 | GitHub: dauparas/ProteinMPNN | Dauparas J, et al. *Science*, 2022 |
| **LigandMPNN** | 配体结合位点序列设计 | GNN，显式建模配体环境 | 酶设计、传感器设计、药物相互作用位点设计 | GitHub: dauparas/LigandMPNN | Dauparas J, et al. *Nature Methods*, 2025 |

### A. 序列设计工具(逆向折叠)

这些工具以蛋白质骨架结构为输入，设计出可能折叠成该结构的氨基酸序列，通常被称为"逆向折叠问题"。

#### 1. ProteinMPNN

**概述**：ProteinMPNN是一种基于深度学习的稳健工具，用于快速蛋白质序列设计，由华盛顿大学蛋白质设计研究所(IPD, David Baker实验室)开发。

**核心功能与设计原理**：
- 利用图神经网络(GNN)对蛋白质骨架及其周围化学环境进行建模
- 基于每个残基的局部原子环境进行条件判断
- 速度快(每个结构约一秒钟完成)
- 无需专家定制
- 在实验室测试中优于先前的最佳工具

**应用场景**：
- 从头蛋白质设计
- 为提高稳定性或功能而重新设计现有蛋白质
- 为给定折叠生成序列库

**使用方法**：
```bash
# 关键参数示例
--model_name vanilla          # 指定模型检查点
--sampling_temp 0.1          # 控制序列多样性(0.0-1.0)
--num_seq_per_target 8       # 每个目标生成的序列数量
--backbone_noise 0.2         # 向骨架原子添加噪声以增强鲁棒性
```

#### 2. LigandMPNN

**概述**：LigandMPNN是ProteinMPNN的扩展，擅长创建与多种非蛋白质分子(小分子、核酸、金属)相互作用的蛋白质。

**核心功能与设计原理**：
- 在ProteinMPNN基础上，显式地将非蛋白质原子(配体)的空间和化学环境信息整合到序列设计过程中
- 编码邻近配体原子、元素类型，并预测目标配体周围的侧链构象
- 在与配体相互作用的残基的序列恢复率方面有显著提高(10–40%)

**应用场景**：
- 设计酶、传感器
- 与特定小分子结合的治疗性蛋白质
- DNA/RNA结合蛋白
- 金属蛋白

### B. 结构生成工具(从头设计)

这些工具旨在从头开始或在某些基序或特性的条件下生成新的蛋白质骨架结构。

#### 1. RFdiffusion

**概述**：RFdiffusion是一种用于从头蛋白质结构设计的生成式AI模型，其灵感来源于AI图像生成器，由IPD开发。

**核心功能与设计原理**：
- 利用基于扩散的深度学习模型将离散的原子云塑造成新的蛋白质骨架
- 学习一个去噪过程来生成结构
- 可在数秒内生成新的蛋白质结构

**主要特性**：
- 支持单体、寡聚体、结合剂等多种类型
- 可执行无条件生成，也可基于基序进行条件生成
- 支持对称寡聚体生成(环状、二面体、四面体)

**使用方法**：
```bash
# Contig字符串示例
'[100-150]'              # 无条件生成，长度100-150
'A10-25/0-0/B5-30'      # 支架化基序A10-25和B5-30并在其间断开链
```

#### 2. Protpardelle

**概述**：Protpardelle是一种蛋白质结构的全原子扩散模型，可同时协同设计骨架、序列和侧链。

**核心功能与设计原理**：
- 将所有侧链状态同时表示为一个"叠加态"
- 在样本生成过程中塌缩为单个残基类型和构象
- 生成具有连贯序列和全原子结构的完整蛋白质

**应用场景**：
- 从头全原子蛋白质设计
- 基于侧链化学性质的功能基序支架化
- 设计序列通过侧链影响骨架构象的蛋白质

### C. 专业化蛋白质/肽设计工具

#### 1. EvoPro (Kuhlman Lab)

**概述**：EvoPro是一个基于遗传算法的蛋白质结合剂优化流程。

**核心功能与设计原理**：
- 在遗传算法框架内使用深度学习模型(AF2用于结构预测，ProteinMPNN用于序列优化)
- 进行紧密蛋白质结合剂的"计算机内进化"
- 支持多状态设计

**应用场景**：
- 设计新的蛋白质结合剂
- 优化现有结合剂以提高亲和力或特异性
- 抗体工程

#### 2. PepPrCLIP (Duke University)

**概述**：PepPrCLIP是一个基于AI的平台，可设计能够结合并摧毁先前"不可成药"的致病蛋白(特别是无序蛋白)的短肽。

**核心功能与设计原理**：
- 受OpenAI的CLIP模型启发
- 使用在天然蛋白质序列上训练的生成算法(PepPr)设计"引导"肽
- 使用改编的CLIP算法根据序列单独筛选这些肽以匹配其靶蛋白

**应用场景**：
- 针对具有挑战性的靶点(如与癌症和其他疾病相关的内在无序蛋白)设计治疗性肽

## III. 蛋白质设计流程中的基本辅助工具

蛋白质设计很少是一个单一步骤的过程，它依赖于一套辅助工具进行结构预测、模拟、分析和可视化。

### 蛋白质设计基本辅助工具概览

| 工具类别 | 工具名称 | 在蛋白质设计流程中的主要作用 | 可用性 |
|---------|---------|----------------------------|--------|
| **结构预测** | ColabFold | 便捷使用AlphaFold2/ESMFold进行高精度预测 | Google Colab, GitHub |
| **结构预测** | ESMFold | 基于pLM的快速单序列结构预测 | HuggingFace, API |
| **MD模拟** | GROMACS | 评估设计蛋白的稳定性、柔性、构象变化 | www.gromacs.org |
| **MD模拟** | OpenMM | 灵活的MD模拟平台，支持自定义力场/流程 | openmm.org |
| **分子对接** | AutoDock Vina | 评估设计蛋白与配体的结合模式和亲和力 | GitHub: ccsb-scripps |
| **可视化** | PyMOL | 检查设计模型、分析相互作用、准备出版图表 | 开源版本可用 |
| **可视化** | ChimeraX | 高级可视化与分析，集成AlphaFold/ESMFold获取 | UCSF官网 |

### A. 蛋白质结构预测

高精度结构预测工具是现代蛋白质设计的基础，它们：
- 为设计提供起始模型
- 允许对设计序列进行计算机内的验证
- 提供了训练新设计算法的庞大数据集

#### 1. ColabFold (用于AlphaFold2 & ESMFold的便捷使用)

**概述**：ColabFold通过Google Colaboratory和命令行工具使AlphaFold2和ESMFold易于使用，缩短了周转时间。

**与设计的相关性**：为没有大量计算资源的研究人员提供易于访问的高精度结构预测，这对于生成模板结构或验证设计至关重要。

#### 2. ESMFold (Meta AI)

**概述**：ESMFold是一种基于ESM-2蛋白质语言模型的端到端单序列3D结构预测器。

**与设计的相关性**：提供非常快速的结构预测，无需多序列比对(MSA)，可用于快速筛选或在难以获得MSA时使用。

### B. 分子动力学(MD)与模拟

MD工具对于评估设计蛋白质的动态行为、稳定性以及与配体或其他蛋白质的相互作用至关重要。

#### 1. GROMACS

**概述**：GROMACS是一款广泛使用、功能多样且快速的开源MD模拟软件包，尤其适用于生物分子。

**使用工作流程**：
```bash
# 典型的GROMACS工作流程
pdb2gmx          # 生成蛋白质拓扑
editconf         # 定义盒子
gmx solvate      # 溶剂化
gmx genion       # 添加离子
# 能量最小化 → NVT平衡 → NPT平衡 → 生产MD
```

#### 2. OpenMM

**概述**：OpenMM是一个高性能、可定制、开源的分子模拟工具包，具有广泛的Python绑定。

**与设计的相关性**：灵活的平台，用于设置和运行设计蛋白质的模拟，特别适用于自定义力场或协议。

### C. 分子对接

#### AutoDock Vina (v1.2.x)

**概述**：AutoDock Vina是一款广泛使用的开源分子对接程序。

**与设计的相关性**：评估设计的蛋白质(受体)与特定配体的结合情况。

**使用方法**：
```bash
# 基本对接命令
vina --receptor receptor.pdbqt \
     --ligand ligand.pdbqt \
     --center_x 25.0 --center_y 25.0 --center_z 25.0 \
     --size_x 20.0 --size_y 20.0 --size_z 20.0 \
     --exhaustiveness 8
```

### D. 可视化软件

#### 1. PyMOL (开源版本)

**概述**：PyMOL是一款广泛使用的分子可视化系统，具有基于Python的命令行。

**与设计的相关性**：
- 可视化设计的蛋白质结构、突变、界面、配体结合
- 创建出版质量的图像和动画
- 分析相互作用和结构特征

#### 2. UCSF ChimeraX

**概述**：ChimeraX是RBVI推出的下一代分子可视化程序，是Chimera的继任者。

**与设计的相关性**：
- 先进的可视化和分析功能
- 集成AlphaFold/ESMFold获取
- 界面分析(碰撞、接触、氢键)
- 对接结果查看

## IV. 集成蛋白质设计工作流程的概念化

### A. 迭代设计的一般原则

蛋白质工程中的**"设计-构建-测试-学习"循环**是核心方法论。计算工具通过在实验验证之前实现计算机内的快速迭代，极大地加速了这一循环。

成功的蛋白质设计依赖于一个迭代循环：
1. **生成设计方案**
2. **预测其特性**(尤其是结构)
3. **验证这些预测**(计算机模拟 → 实验验证)

### B. 示例工作流程1：从头酶设计

**目标**：设计一种新型酶来催化特定反应。

**步骤与工具集成**：

1. **活性位点定义**
   - 定义期望的催化几何结构
   - 确定关键催化残基的位置、过渡态类似物

2. **骨架生成**
   - 使用 **RFdiffusion** 或 **Protpardelle** 在活性位点基序的条件下生成新的蛋白质骨架
   - 如果涉及辅因子，可使用 **RFdiffusion All-Atom**

3. **序列设计**
   - 如果包含过渡态类似物或辅因子：使用 **LigandMPNN**
   - 否则使用 **ProteinMPNN** 为生成的骨架设计序列

4. **结构预测与初步筛选**
   - 使用 **ColabFold (AlphaFold2/ESMFold)** 或 **RoseTTAFold All-Atom** 预测设计序列的结构
   - 检查它们是否按预期折叠以及活性位点几何结构是否保持

5. **对接(底物/产物)**
   - 使用 **AutoDock Vina** 或 **rDock** 对接底物和产物
   - 评估结合相容性

6. **MD模拟**
   - 使用 **GROMACS** 或 **OpenMM** 模拟有希望的候选物
   - 评估活性位点稳定性、柔性和与底物/辅因子的相互作用

7. **可视化与分析**
   - 在整个过程中使用 **PyMOL** 或 **ChimeraX** 进行检查和分析

8. **迭代**
   - 根据计算机模拟评估结果，优化活性位点、骨架或序列

### C. 示例工作流程2：治疗性抗体/结合剂设计

**目标**：设计一种与靶蛋白高亲和力结合的结合剂。

**步骤与工具集成**：

1. **靶标结构**
   - 获取靶蛋白结构(实验或由 **AlphaFold2/RoseTTAFold** 预测)

2. **初始结合剂生成/优化**
   - 抗体样结合剂：使用 **EvoPro**
   - 针对无序靶标的肽：使用 **PepPrCLIP**
   - D-肽：使用 **DexDesign (OSPREY)**

3. **界面序列设计**
   - 使用 **LigandMPNN** 设计与靶标结合的界面残基

4. **复合物结构预测**
   - 使用 **RoseTTAFold All-Atom** 或 **AlphaFold-Multimer** 预测复合物结构

5. **MD模拟**
   - 模拟复合物，评估结合稳定性、界面动力学

6. **可视化与分析**
   - 使用 **PyMOL/ChimeraX** 检查结合界面、接触、氢键

## V. 结论与未来展望

### A. 当前格局总结

自2021年以来，开源蛋白质设计工具领域取得了显著进展，尤其受到AI和深度学习的深刻影响。目前，研究人员可以利用一系列强大的非商业化工具进行：

- **序列设计**
- **结构生成**  
- **特定功能设计**
- **结构预测**
- **动态模拟**
- **结果分析**

### B. 新兴趋势与挑战

**新兴趋势**：
- **面向功能的设计**：从仅仅关注结构转向直接设计特定功能
- **多约束优化**：设计同时满足多种标准的蛋白质
- **处理非经典氨基酸**：将设计能力扩展到20种标准氨基酸之外
- **改进生成模型**：提高可控性、减少偏差、提高采样效率

**主要挑战**：
- **实验验证瓶颈**：计算设计正在加速，但实验验证仍然是速率限制步骤
- **深度学习模型的可解释性**：理解DL模型为何做出某些设计选择仍然是挑战
- **专业知识要求**：有效应用这些工具需要深厚的专业知识

### C. 未来展望

未来预期发展方向：

1. **AI与物理原理的进一步融合**
2. **计算设计与高通量自动化实验的紧密结合**
3. **从"折叠设计"向"功能设计"的持续演进**
4. **更好的工具集成和工作流程自动化**

计算蛋白质设计有望在医药、材料科学和可持续发展等领域应对重大的社会挑战。随着工具的不断完善和普及，这一领域必将在解决人类面临的重大问题中发挥越来越重要的作用。
